* vi-find-char
A lightweight Emacs package that emulates Vi's =f= command for quick
character navigation.

** Overview
=vi-find-char= provides a fast way to jump to the next or previous
occurrence of any character in your buffer. Press the forward key (default
=C-.=) or backward key (default =C-,=), type a character, and point jumps
immediately.  Press the same key again to continue searching in that
direction without being prompted again.  Both keys are configurable; see
the Customization section.

** Features
- *Two keystrokes*: Jump to target character with two keystrokes
- *Configurable keybindings*: Default =C-.= for forward search, =C-,= for
  backward search; customize via =vi-find-char-forward-key= and
  =vi-find-char-backward-key=
- *Smart repeat*: Press the forward or backward key again to repeat the last search
- *Direction switching*: Alternate between forward and backward
  seamlessly

** Installation
*** Manual Installation
1. Download =vi-find-char.el= to your Emacs load path
2. Add to your init file:

#+begin_src elisp
(require 'vi-find-char)
#+end_src

*** Using =use-package=
#+begin_src elisp
(use-package vi-find-char
  :load-path "path/to/vi-find-char")
#+end_src

** Usage
The examples below use the default keys (=C-.= for forward, =C-,= for backward).

*** Basic Search
- Press =C-.= to search forward for a character
- At the prompt "Find forward:", type any character
- Point jumps to the next occurrence of that character
- Press =C-g= to cancel at any time

*** Backward Search
- Press =C-,= to search backward for a character
- At the prompt "Find backward:", type any character
- Point jumps to the previous occurrence of that character

*** Repeat Search
- Press the forward or backward key to bring up the prompt
- At the prompt, press the forward or backward key again to repeat:
  - Forward key repeats the last search forward
  - Backward key repeats the last search backward
- You can alternate between forward and backward seamlessly
- Plain =.= and =,= characters can be searched for normally (when those
  are not your configured keys)

*** Examples
#+begin_example
Text: "The quick brown fox jumps over the lazy dog"
      ^
      (point is here)

1. Press C-.
2. Type 'o'
   → Point moves to "brown f[o]x"

3. Press C-. (brings up prompt), then press C-. again to repeat
   → Point moves to "f[o]x jumps"

4. Press C-. (brings up prompt), then press C-, to repeat backward
   → Point moves back to "brown f[o]x"

5. To search for a period: Press C-., then type '.'
   → Searches for the '.' character in the text
#+end_example

** Customization
*** Custom Keybindings
If you prefer different keybindings, customize
=vi-find-char-forward-key= and =vi-find-char-backward-key=:

#+begin_src elisp
(use-package vi-find-char
  :load-path "path/to/vi-find-char"
  :custom
  (vi-find-char-forward-key  "M-.")
  (vi-find-char-backward-key "M-,"))
#+end_src

Or after loading:

#+begin_src elisp
(setopt vi-find-char-forward-key  "M-.")
(setopt vi-find-char-backward-key "M-,")
#+end_src

You can also use =M-x customize-group RET vi-find-char RET= to change
the keys interactively.  The old binding is removed automatically when
you apply a new key.

*** Single-stroke keys required
This package only supports single-event key sequences (one keystroke, no
chords like =C-c .=).  This is a fundamental constraint: the repeat
detection at the prompt reads one key at a time via =read-key=.

This also means *finding a suitable key pair is the hardest part of using
this package*.  If you cannot bind both the forward and backward commands
to single, easy-to-reach keys, the package offers no advantage over
=C-s/C-r= (isearch).  The default =C-.= and =C-,= conflict with
=flyspell-mode=; other common single-stroke keys may conflict with
=iedit=, =xref=, or major-mode bindings.

Suggested pairs to evaluate against your own setup:

| Forward | Backward | Common conflict      |
|---------+----------+----------------------|
| =C-.=   | =C-,=    | =flyspell-mode=      |
| =C-'=   | =C-;=    | =C-;= is =iedit=     |
| =C-'=   | =M-'=    | usually free         |

** Comparison with Similar Features
*** vs. =C-s <char> RET=
- *isearch*: =C-s <char> RET=
- *vi-find-char*: =C-. <char>= (no need to press RET)
- Saves one keystroke per search

*** vs. Vi's =f= command
- *Vi*: =f<char>= finds forward, =;= repeats, =,= reverses
- *vi-find-char*: =C-.<char>= finds forward, =C-.= repeats, =C-,=
  reverses
- More Emacs-like with explicit forward/backward keys

*** vs. =avy=, =ace-jump-mode=
- Those packages provide visual overlays for multi-step jumps
- =vi-find-char= is simpler: just type the character and go
- Better for nearby targets; =avy= better for distant targets

** Implementation Details
The package uses a two-layer architecture:

1. *Entry commands* (=vi-find-char-go-forward=, =vi-find-char-go-backword=)
   prompt for input using =read-key=.  At the prompt:
   - Pressing the configured forward or backward key repeats the last search
   - Any other character triggers a new search for that character
   - =C-g= cancels (handled via =condition-case=)
2. *Search core* (=vi-find-char-search=) performs =search-forward= or
   =search-backward= based on the direction flag, and deactivates the mark
   on success.

State is maintained in two variables:
- =vi-find-char-last-char=: the last searched character (global, for repeats)
- =vi-find-char-forward=: current search direction (buffer-local)
