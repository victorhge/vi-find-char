* vi-find-char
This is a Emacs package that emulates Vi's =f= command for quick character
navigation, with two keystrokes.

Press the forward key (default =C-.=) or backward key (default =C-,=), type a
character, and point jumps immediately.  Press the same key again to continue
searching in that direction without being prompted again.  Both keys are
configurable; see the Customization section.

** Features
- *Two keystrokes*: Jump to target character with two keystrokes
- *Configurable keybindings*: Default =C-.= for forward search, =C-,= for
  backward search; customize via =vi-find-char-forward-key= and
  =vi-find-char-backward-key=
- *Smart repeat*: Press the forward or backward key again to repeat the last search
- *Direction switching*: Alternate between forward and backward seamlessly
- *Visual flash*: Matched character and other occurrences on the line briefly
  flash after each jump; duration and region are configurable

** Installation
*** Manual Installation
1. Download =vi-find-char.el= to your Emacs load path
2. Add to your init file:

#+begin_src elisp
(require 'vi-find-char)
#+end_src

*** Using =use-package=
#+begin_src elisp
(use-package vi-find-char
  :load-path "path/to/vi-find-char")
#+end_src

** Customization
*** Visual Flash
After each successful jump, the matched character flashes with =vi-find-char-match-face=
(inherits =isearch=) and other occurrences of that character on the same line flash
with =vi-find-char-other-match-face= (inherits =lazy-highlight=).  A message
is also shown in the minibuffer (e.g. =Found 'x'=).

Two options control this behaviour:

| Variable                      | Type    | Default | Description                                      |
|-------------------------------+---------+---------+--------------------------------------------------|
| =vi-find-char-flash-duration= | float   | =0.5=   | Seconds before highlights clear; =0= disables   |
| =vi-find-char-flash-lines=    | integer | =0=     | Lines above/below the match to include           |

Example — extend the flash region to 2 lines above and below, keep for 1 second:

#+begin_src elisp
(setopt vi-find-char-flash-duration 1.0)
(setopt vi-find-char-flash-lines    2)
#+end_src

To disable the flash entirely:

#+begin_src elisp
(setopt vi-find-char-flash-duration 0)
#+end_src

*** Custom Keybindings
If you prefer different keybindings, customize
=vi-find-char-forward-key= and =vi-find-char-backward-key=:

#+begin_src elisp
(use-package vi-find-char
  :load-path "path/to/vi-find-char"
  :custom
  (vi-find-char-forward-key  "M-.")
  (vi-find-char-backward-key "M-,"))
#+end_src

Or after loading:

#+begin_src elisp
(setopt vi-find-char-forward-key  "M-.")
(setopt vi-find-char-backward-key "M-,")
#+end_src

You can also use =M-x customize-group RET vi-find-char RET= to change
the keys interactively.  The old binding is removed automatically when
you apply a new key.

*** Single-stroke keys required
This package only supports single-event key sequences (one keystroke, no
chords like =C-c .=).  This is a fundamental constraint: the repeat
detection at the prompt reads one key at a time via =read-key=.

This also means *finding a suitable key pair is the hardest part of using
this package*.  If you cannot bind both the forward and backward commands
to single, easy-to-reach keys, the package offers no advantage over
=C-s/C-r= (isearch).  The default =C-.= and =C-,= conflict with
=flyspell-mode=; other common single-stroke keys may conflict with
=iedit=, =xref=, or major-mode bindings.

Suggested pairs to evaluate against your own setup:

| Forward | Backward | Common conflict      |
|---------+----------+----------------------|
| =C-.=   | =C-,=    | =flyspell-mode=      |
| =C-'=   | =C-;=    | =C-;= is =iedit=     |
| =C-'=   | =M-'=    | usually free         |

** Comparison with Similar Features
*** vs. =C-s <char> RET=
- *isearch*: =C-s <char> RET=
- *vi-find-char*: =C-. <char>= (no need to press RET)
- Saves one keystroke per search

*** vs. Vi's =f= command
- *Vi*: =f<char>= finds forward, =;= repeats, =,= reverses
- *vi-find-char*: =C-.<char>= finds forward, =C-.= repeats, =C-,=
  reverses
- More Emacs-like with explicit forward/backward keys

*** vs. =avy=, =ace-jump-mode=
- Those packages provide visual overlays for multi-step jumps
- =vi-find-char= is simpler: just type the character and go
- Better for nearby targets; =avy= better for distant targets

** Implementation Details
The package uses a two-layer architecture:

1. *Entry commands* (=vi-find-char-go-forward=, =vi-find-char-go-backward=)
   prompt for input using =read-key=.  At the prompt:
   - Pressing the configured forward or backward key repeats the last search
   - Any other character triggers a new search for that character
   - =C-g= cancels (handled via =condition-case=)
2. *Search core* (=vi-find-char--search=) performs =search-forward= or
   =search-backward=.  On success it flashes the result via
   =vi-find-char--flash= and echoes =Found 'x'=.  On failure it echoes
   =Character 'x' not found=.

State is maintained in one variable:
- =vi-find-char-last-char=: the last searched character (buffer-local, for repeats)
