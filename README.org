* vi-find-char
A lightweight Emacs package that emulates Vi's =f= command for quick
character navigation.

** Overview
=vi-find-char= provides a fast way to jump to the next or previous
occurrence of any character in your buffer. Press =C-.= or =C-,=, type a
character, and point jumps immediately. Press =C-.= or `C-,' twice to
continue searching in that direction without being prompted again.

** Features
- *Simple keybindings*: =C-.= for forward search, =C-,= for backward
  search
- *Minibuffer prompts*: Clear visual feedback showing search direction
- *Smart repeat*: Press =C-.= or =C-,= again to repeat the last search
- *Direction switching*: Alternate between forward and backward
  seamlessly
- *Mark deactivation*: Automatically deactivates the mark after
  successful search
- *Lightweight*: Single file, no dependencies, ~100 lines of code

** Installation
*** Manual Installation
1. Download =vi-find-char.el= to your Emacs load path
2. Add to your init file:

#+begin_src elisp
(require 'vi-find-char)
#+end_src

*** Using =use-package=
#+begin_src elisp
(use-package vi-find-char
  :load-path "path/to/vi-find-char")
#+end_src

** Usage
*** Basic Search
- Press =C-.= to search forward for a character
- At the prompt "Find forward:", type any character
- Point jumps to the next occurrence of that character
- Press =C-g= to cancel at any time

*** Backward Search
- Press =C-,= to search backward for a character
- At the prompt "Find backward:", type any character
- Point jumps to the previous occurrence of that character

*** Repeat Search
- Press =C-.= or =C-,= to bring up the prompt
- At the prompt, press =C-.= or =C-,= again to repeat:
  - =C-.= repeats the last search forward
  - =C-,= repeats the last search backward
- You can alternate between forward and backward seamlessly
- Plain =.= and =,= characters can be searched for normally

*** Examples
#+begin_example
Text: "The quick brown fox jumps over the lazy dog"
      ^
      (point is here)

1. Press C-.
2. Type 'o'
   → Point moves to "brown f[o]x"

3. Press C-. (brings up prompt), then press C-. again to repeat
   → Point moves to "f[o]x jumps"

4. Press C-. (brings up prompt), then press C-, to repeat backward
   → Point moves back to "brown f[o]x"

5. To search for a period: Press C-., then type '.'
   → Searches for the '.' character in the text
#+end_example

** Keybindings
| Key   | Command                    | Description                |
|-------+----------------------------+----------------------------|
| =C-.= | =vi-find-char-go-forward=  | Search forward for a char  |
| =C-,= | =vi-find-char-go-backword= | Search backward for a char |

*Note*: These keybindings are set globally when the package is loaded.

** Customization
*** Custom Keybindings
If you prefer different keybindings, you can change them in your init
file:

#+begin_src elisp
(require 'vi-find-char)

;; Unbind default keys
(keymap-global-unset "C-.")
(keymap-global-unset "C-,")

;; Set your preferred keys
(keymap-global-set "M-f" 'vi-find-char-go-forward)
(keymap-global-set "M-b" 'vi-find-char-go-backword)
#+end_src

** Comparison with Similar Features
*** vs. =C-s <char> RET=
- *vi-find-char*: =C-. <char>= (no need to press RET)
- *isearch*: =C-s <char> RET=
- Saves one keystroke per search

*** vs. Vi's =f= command
- *Vi*: =f<char>= finds forward, =;= repeats, =,= reverses
- *vi-find-char*: =C-.<char>= finds forward, =C-.= repeats, =C-,=
  reverses
- More Emacs-like with explicit forward/backward keys

*** vs. =avy=, =ace-jump-mode=
- Those packages provide visual overlays for multi-step jumps
- =vi-find-char= is simpler: just type the character and go
- Better for nearby targets; =avy= better for distant targets

** Implementation Details
The package uses a two-layer architecture:

1. *Entry commands* check if the previous command was a vi-find-char
   command (via =last-command=)
   - If yes: repeat the last search in the current direction
   - If no: prompt for a new character using =read-char=
2. *Search core* performs =search-forward= or =search-backward= based on
   the direction flag

State is maintained in two variables: - =vi-find-char-last-char=: The
last searched character (for repeats) - =vi-find-char-forward=: Current
search direction (buffer-local)
